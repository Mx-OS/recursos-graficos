Manual de Identidad Visual MxOS

Introducción
============

**MxOS** es una distribución Linux segura, abierta y soberana para México. Ideal para el sector público, empresas, instituciones educativas y usuarios que buscan una alternativa Linux robusta, gratuita y adaptada a nuestro entorno. Este manual establece las bases para el uso coherente de su identidad visual.

Tipografía
==========

La identidad visual de MxOS establece un sistema tipográfico claro y accesible que acompaña la comunicación gráfica del proyecto. Las fuentes seleccionadas son de código abierto y están disponibles para su uso libre, lo cual permite mantener coherencia visual en todos los medios.

Fuente principal
----------------

**Hack** es la fuente tipográfica oficial principal para todos los materiales relacionados con código fuente, documentación técnica o interfaces que lo requieran.

- Sitio del proyecto: `Hack en GitHub <https://github.com/source-foundry/Hack>`__

Hack está diseñada para ser una fuente de trabajo infalible para código fuente. Tiene un profundo arraigo en la comunidad tipográfica de código abierto y se basa en las contribuciones de los proyectos Bitstream Vera y DejaVu. Su diseño de gran altura x, amplia apertura y bajo contraste la hacen legible en tamaños de texto de código fuente comunes, con un rango óptimo de 8 a 14 caracteres. El conjunto completo de cambios al código fuente original está disponible en el registro de cambios.

Fuentes secundarias
-------------------

**Fira Sans**

- Diseñada para ofrecer buena legibilidad en pantalla y papel.
- Ideal para títulos, subtítulos y textos corridos.
- De uso libre y código abierto.

**Roboto**

- Fuente versátil, moderna y neutral.
- Diseñada especialmente para interfaces digitales y dispositivos móviles.
- Su estructura geométrica con trazos suaves la hace ideal para comunicar tecnología y simplicidad.

.. note::

   Todas las fuentes mencionadas están disponibles de forma gratuita y pueden ser utilizadas sin restricciones comerciales. Se recomienda no sustituirlas por otras fuentes distintas para conservar la integridad visual del proyecto MxOS.


Tono de Comunicación
====================

El lenguaje utilizado en las comunicaciones de MxOS debe ser:

- **Formal pero accesible**: Claro y directo, sin tecnicismos innecesarios.
- **Inclusivo**: Evitar sesgos de género y promover la diversidad.
- **Centrado en la soberanía tecnológica**: Resaltar la importancia de la autonomía digital.
